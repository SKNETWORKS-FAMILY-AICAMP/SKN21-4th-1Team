# 구현 계획 - 시연 자동화 스크립트

이 계획의 목표는 시연 영상을 위해 브라우저 내에서 현실적인 "고스트 유저(Ghost User)" 시뮬레이션을 만드는 것입니다. 이를 통해 외부 자동화 도구의 "파란 테두리"나 "블러" 현상을 방지합니다.

## 사용자 검토 필요
> [!IMPORTANT]
> **삭제 기능 부재**: `chat.html`이나 `views.py`에서 기존 "채팅 삭제" 버튼을 찾을 수 없었습니다. 시나리오상의 "채팅 삭제" 단계를 수행하기 위해 **UI에 삭제 버튼을 추가**하겠습니다.
> - 백엔드 구현(실제 삭제)이 필요한가요? 현재는 지시가 없으므로 **UI 전용**(화면에서만 리스트 제거)으로 구현하거나 더미 버튼으로 처리하겠습니다.
> **가정된 시나리오**:
> 1. 메인 채팅에서 시작.
> 2. "새로운 채팅" 클릭.
> 3. 3가지 질문 입력 및 전송.
> 4. 채팅 세션 삭제.
> 5. 로그아웃.

## 변경 제안

### [Frontend] 채팅 인터페이스
#### [MODIFY] [chat.html](chat/templates/chat/chat.html)
- `demo_automation.js` 포함.
- 사이드바 세션 항목에 "삭제" 버튼(휴지통 아이콘) 추가.
- 데모 활성화 시 실제 커서를 숨기는 CSS 추가 (`body.demo-active { cursor: none; }`).

#### [NEW] [demo_automation.js](chat/static/js/demo_automation.js)
- **GhostCursor 클래스**: 가짜 커서 요소(SVG)를 생성하고 위치를 관리합니다(베지에 곡선을 사용하여 자연스러운 움직임 구현).
- **액션**:
    - `moveTo(selector)`: 해당 요소로 이동
    - `click(selector)`: 클릭
    - `type(selector, text)`: 텍스트 입력
    - `wait(ms)`: 대기
- **시나리오 관리**:
    - `sessionStorage`를 사용하여 단계(step) 추적:
    - **Step 0**: 첫 질문(Q1) 전송 -> 답변 대기.
    - **Step 1**: "새로운 채팅" 버튼 클릭 (페이지 리로드).
    - **Step 2**: 이전 채팅 세션(사이드바 두 번째 항목)에 마우스 오버 -> "삭제" 버튼 클릭 -> 모달 확인 클릭.
    - **Step 3**: "로그아웃" 클릭 -> (확인 페이지 이동 시) 확인 버튼 클릭.

## 검증 계획

### 수동 검증
1.  **데모 모드 활성화**: 특정 URL 파라미터 `?demo=true` 또는 콘솔 명령어 `startDemo()`.
2.  **관찰**:
    -   실제 커서가 사라져야 함.
    -   가짜 커서가 나타나야 함.
    -   마우스가 "새로운 채팅"으로 부드럽게 이동해야 함.
    -   페이지가 새로고침되어야 함(새 채팅).
    -   스크립트가 재개되어 -> "질문 1" 입력 -> 전송.
    -   ...
    -   마지막으로 로그아웃.
3.  **녹화**: 사용자가 OBS/QuickTime 등을 사용하여 이 과정이 실행되는 동안 화면을 녹화.
